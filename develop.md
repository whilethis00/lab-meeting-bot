## 구현 완료 기능

### 스마트 화자 자동 매핑 ✅

**개요**: 회의록 업로드 시 미매핑 화자가 있으면 봇이 먼저 참여자를 물어보고, 발언 내용을 분석해 자동으로 이름을 매핑.

**동작 흐름**:
```
1. 파일 업로드
   ↓
2. "👥 4명의 화자가 감지됐습니다! 참여자 이름을 알려주세요."
   ↓
3. 사용자: "나, 교수님, 찬우님, 성빈님"
   ↓
4. LLM이 각 화자의 발언 패턴 분석 →
   • Speaker 1 → 교수님 (질문/방향 제시 발언 패턴)
   • Speaker 2 → 나 (발표/보고 발언 패턴)
   ...
   ↓
5. 매핑 저장 후 분석 파이프라인 자동 재개
```

**구현 포인트**:
- `_pending_name_input` dict로 대화 상태 관리 (chat_id별)
- `SMART_SPEAKER_ASSIGN_PROMPT`: 발언 샘플 3개 + 이름 목록 → LLM이 추론
- 매핑 완료 즉시 요약/할일 추출 파이프라인 자동 재개

## 1순위. 다국어 요약 출력

**아이디어**: 회의록 분석 결과를 한국어 외에 영어·중국어 등 원하는 언어로 동시 출력.
영어 회의록을 입력해도 한국어로 요약 출력 가능.

**평가**
- 구현 용이성 ★★★★★ — 프롬프트에 언어 지시만 추가하면 됨
- 발표 임팩트 ★★★★☆ — "다국적 연구실"에서 즉시 유용하다는 걸 시연하기 쉬움
- 실용성 ★★★★☆ — 영어 논문 미팅, 외국인 협업 시 즉시 활용

**구현 방법**
1. `/lang` 명령어 또는 `/setlang ko+en` 식으로 출력 언어 설정
2. `chat_context` 테이블 또는 `.env`에 `OUTPUT_LANG` 저장
3. `SUMMARIZER_PROMPT` 및 `ACTION_EXTRACTOR_PROMPT`에 `{lang}` 변수 추가
4. 요약 결과를 설정된 언어로 병렬 생성 후 구분선으로 묶어 전송

```
# 텔레그램 명령어
/setlang ko en       → 한국어 + 영어 동시 출력
/setlang ko cn       → 한국어 + 중국어 동시 출력
```

## 2순위. 교수님 피드백 자동 기록

**아이디어**: 회의록에서 교수님(특정 화자)의 발언 중 피드백·지적·제안 패턴을 추출해 별도로 누적 저장.
"교수님이 자주 지적하시는 게 뭐야?"라고 물으면 패턴 분석 결과를 반환.

**평가**
- 구현 용이성 ★★★☆☆ — DB 테이블 추가 + 프롬프트 설계 필요
- 발표 임팩트 ★★★★★ — "AI가 교수님 피드백을 학습한다"는 스토리가 강력
- 실용성 ★★★★★ — 반복 지적 사항을 미리 파악하고 대비 가능

**구현 방법**
1. DB에 `professor_feedback` 테이블 추가 (`meeting_id`, `speaker`, `content`, `category`, `date`)
2. `action_extractor`와 별도로 `feedback_extractor` 에이전트 추가
   - 카테고리: `methodology` / `writing` / `presentation` / `experiment` / `other`
3. 회의록 처리 파이프라인 마지막 단계에서 교수 발언 분리 → 피드백 추출 → 저장
4. `/feedback` 명령어로 카테고리별 빈도·최근 피드백 조회

```
사용자: 교수님이 자주 뭐라고 하셨어?
봇: 📊 교수님 피드백 패턴 (총 12회의 분석)
    🔴 실험 설계 [5회]: "비교군 설정이 명확해야..."
    🟡 논문 작성 [4회]: "관련 연구 보강 필요..."
    🟢 발표 방식 [3회]: "결론을 먼저 말하고..."
```

## 3순위. 회의 통계 리포트

**아이디어**: 누적된 회의 데이터를 기반으로 발언 비율, 주제 분포, 할 일 완료율 등 통계를 텍스트 차트로 제공.

**평가**
- 구현 용이성 ★★★☆☆ — DB 집계 쿼리 + 텍스트 시각화
- 발표 임팩트 ★★★★☆ — 데이터 누적 효과를 눈으로 보여줄 수 있음
- 실용성 ★★★☆☆ — 회의가 쌓일수록 가치 증가

**구현 방법**
1. `/stats` 명령어 추가
2. `queries.py`에 집계 쿼리 추가 (화자별 발언 세그먼트 수, 주제 분포, action item 완료율)
3. 텍스트 기반 막대 그래프로 포맷팅 (외부 라이브러리 없이 구현 가능)

```
/stats →
📊 랩미팅 통계 (최근 4회)
발언 비율: 화자1 ████░░ 42%  화자2 ███░░░ 35%  화자3 ██░░░░ 23%
주요 주제: 실험결과 > 논문작성 > 향후계획
할일 완료율: ████████░░ 78% (14/18건)
```

## 4순위. 할 일 마감 리마인더

**아이디어**: 마감 기한이 설정된 액션 아이템의 D-1, D-day에 자동으로 텔레그램 알림 전송.

**평가**
- 구현 용이성 ★★★☆☆ — 스케줄러(APScheduler) 연동 필요
- 발표 임팩트 ★★★☆☆ — "능동적으로 동작하는 봇"을 보여줄 수 있음
- 실용성 ★★★★☆ — 실제 사용 시 가장 체감되는 기능

**구현 방법**
1. `requirements.txt`에 `APScheduler` 추가
2. `bot/main.py`에 매일 오전 9시 실행되는 스케줄러 등록
3. `queries.py`에 오늘/내일 마감 액션 조회 쿼리 추가
4. 해당 chat_id로 알림 메시지 전송

```
[자동 알림 - 매일 09:00]
⏰ 오늘 마감 할 일
• 🔴 실험 결과 정리 — 김연구원 (오늘까지)
• 🟡 논문 초안 검토 — 박연구원 (오늘까지)
```

## 5순위. 서브지도교수 페르소나 (장기 과제)

**아이디어**: 누적된 교수님 발언·피드백 데이터를 학습시켜, 교수님 말투와 관점을 모사하는 AI 페르소나를 만드는 것.
"내 논문 초안 보고 교수님이라면 뭐라고 하실 것 같아?"라는 질문에 답변 가능.

**평가**
- 구현 용이성 ★★☆☆☆ — Few-shot 프롬프트 엔지니어링 + 충분한 데이터 누적 필요
- 발표 임팩트 ★★★★★ — 발표에서 가장 인상적인 데모가 될 수 있음
- 실용성 ★★★★★ — 실제 연구 활용도 매우 높음
- **하루 내 완성 리스크 높음** — 데이터가 적으면 퀄리티가 낮아 역효과

**구현 방법** (시간이 더 있을 때)
1. 2순위(피드백 기록)가 충분히 쌓인 후 진행 권장 (최소 5회 이상 회의)
2. 교수님 발언 전체를 Few-shot 예시로 구성한 시스템 프롬프트 생성
3. `/ask_prof 논문 초안 검토해줘` 명령어로 교수님 스타일 피드백 생성
4. 실제 피드백과 AI 피드백 비교 기능 추가 시 데모 효과 극대화

## 하루 내 추천 개발 순서

```
오전 (3h): 1순위 다국어 기능 구현 + 테스트
오후 (4h): 2순위 교수님 피드백 기록 구현
저녁 (2h): 3순위 또는 4순위 중 1개 선택 구현
           → 데모 스크립트 작성 및 발표 준비
```

> 5순위(서브지도교수)는 발표에서 **"향후 발전 방향"** 으로 소개하는 것이 리스크 없이 임팩트를 줄 수 있는 전략.

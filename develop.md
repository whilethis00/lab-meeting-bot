# Lab Meeting Bot — 기능 개발 계획

---

## 구현 완료 기능

### ✅ 다글로 포맷 지원
클로바노트(`화자 N HH:MM:SS`) 외에 다글로(`MM:SS Speaker N`) 포맷도 자동 감지·파싱.
두 포맷 중 매칭 수가 더 많은 쪽을 자동 선택.

---

### ✅ 스마트 화자 자동 매핑

**개요**: 회의록 업로드 시 미매핑 화자가 있으면 참여자 이름을 물어보고, 각 화자의 발언 패턴을 LLM(Claude Haiku)이 분석해 자동으로 이름을 배정.

**동작 흐름**:
```
파일 업로드
  ↓
"👥 4명의 화자 감지! 참여자 이름을 알려주세요."
  ↓
사용자: "나, 교수님, 찬우님, 성빈님"
  ↓
LLM이 화자별 발언 샘플 분석:
  • Speaker 1 → 교수님 (질문·방향 제시 패턴)
  • Speaker 2 → 나 (발표·보고 패턴)
  ...
  ↓
매핑 저장 → 분석 파이프라인 자동 재개
```

**구현 포인트**:
- `_pending_name_input` dict로 chat_id별 대화 상태 관리
- `SMART_SPEAKER_ASSIGN_PROMPT`: 화자별 발언 샘플 3개 + 이름 목록 → LLM 추론 + 이유 반환
- 모델: Claude Haiku (빠르고 저렴)

---

### ✅ 다국어 요약 출력

**개요**: 회의록 업로드 시 언어 선택 인라인 버튼을 통해 원하는 언어로 요약 출력.
영어 회의록을 올려도 한국어로 요약 가능. `/setlang`으로 기본값 저장 가능.

**지원 언어**: 🇰🇷 한국어 / 🇺🇸 English / 🇨🇳 中文 / 🇯🇵 日本語

**동작 흐름**:
```
화자 매핑 완료
  ↓
┌─────────────────────────┐
│ 🌐 요약 언어를 선택해주세요 │
│ 🇰🇷 한국어  🇺🇸 English   │
│ 🇨🇳 中文    🇯🇵 日本語   │
└─────────────────────────┘
  ↓ 버튼 탭
선택 언어로 요약·할일 출력
```

**구현 포인트**:
- `chat_settings` DB 테이블에 chat_id별 언어 저장
- `SUMMARIZER_PROMPT`에 `{lang_instruction}` 변수 삽입
- Telegram `InlineKeyboardMarkup` + `CallbackQueryHandler`
- `/setlang ko|en|zh|ja`, `/lang` 명령어

## 미구현 기능 (발표 후 개발 예정)

### 6순위. 랩미팅 녹음 알림 & 음성 직접 처리

**배경**: 랩미팅 때마다 녹음 켜는 걸 깜빡하는 문제 → 전사본 자체가 없어서 봇을 못 씀.

---

#### 방향 A. 정기 미팅 알림 (단기, 구현 쉬움)

`/setmeeting` 명령어로 정기 미팅 시간 등록 → APScheduler가 매주 30분·5분 전 자동 텔레그램 알림.

```
/setmeeting 목 14:00
→ 매주 목요일 13:30, 13:55에 자동 알림:
  "📹 30분 후 랩미팅입니다. 지금 녹음 앱 켜세요!"
  "📹 5분 후 랩미팅입니다. 녹음 시작하셨나요?"
```

**구현 방법**
1. `meeting_schedule` 테이블 추가 (`chat_id`, `weekday`, `time`)
2. APScheduler 매분 실행 → 30분·5분 전 해당 chat_id에 알림
3. `/setmeeting`, `/delmeeting`, `/schedule` 명령어

**평가**
- 구현 용이성 ★★★★☆ — APScheduler는 4순위 리마인더와 같은 인프라
- 발표 임팩트 ★★★☆☆ — 데모보다 실용성 포인트
- 실용성 ★★★★★ — 근본 문제(까먹음) 직접 해결

---

#### 방향 B. 음성 직접 처리 (장기, 근본 해결)

다글로·클로바노트 없이 텔레그램 음성메시지를 봇에 직접 보내면 자동 전사 → 분석.

```
[텔레그램에서]
🎙 음성 메시지 전송
  ↓
봇이 Whisper API로 전사
  ↓
기존 분석 파이프라인 (화자 매핑 → 언어 선택 → 요약)
```

**구현 방법**
1. `MessageHandler(filters.VOICE | filters.AUDIO)` 핸들러 추가
2. `openai.Audio.transcribe()` (Whisper API) 또는 `faster-whisper` 로컬 모델로 전사
3. 전사 결과를 기존 `_handle_transcript()`에 전달

**평가**
- 구현 용이성 ★★☆☆☆ — Whisper API 연동 + 오디오 처리 필요
- 발표 임팩트 ★★★★★ — "녹음 앱 필요 없이 바로 텔레그램으로" 시연 효과 강력
- 실용성 ★★★★★ — 워크플로우 단계 2개 제거 (녹음앱 실행 → 전사 → 봇)
- 추가 비용: Whisper API $0.006/분 또는 로컬 모델로 무료 가능

> **추천 전략**: 방향 A를 먼저 구현해 즉시 사용, 방향 B는 장기 목표로 설정.
> 방향 B 구현 시 발표에서 "녹음 앱 없이 바로 텔레그램 한 방에" 데모가 가능해 임팩트 극대화.

### 2순위. 교수님 피드백 자동 기록

**아이디어**: 교수님 발언에서 피드백·지적·제안을 카테고리별로 누적 저장.
"교수님이 자주 뭐라고 하셔?" 물으면 패턴 분석 결과 반환.

**평가**
- 구현 용이성 ★★★☆☆
- 발표 임팩트 ★★★★★ — "AI가 교수님을 분석한다"는 스토리
- 실용성 ★★★★★

**구현 방법**
1. `professor_feedback` 테이블 추가 (`meeting_id`, `speaker`, `content`, `category`, `date`)
2. `feedback_extractor` 에이전트 추가 (카테고리: methodology / writing / presentation / experiment)
3. 파이프라인 마지막에 교수 발언 분리 → 피드백 추출 → 저장
4. `/feedback` 명령어로 카테고리별 빈도·최근 피드백 조회

```
교수님이 자주 뭐라고 하셨어?
→ 📊 교수님 피드백 패턴 (총 12회 분석)
   🔴 실험 설계 [5회]: "비교군 설정이 명확해야..."
   🟡 논문 작성 [4회]: "관련 연구 보강 필요..."
   🟢 발표 방식 [3회]: "결론을 먼저 말하고..."
```

---

### 3순위. 회의 통계 리포트

**아이디어**: 누적 회의 데이터 기반 발언 비율·주제 분포·할일 완료율 텍스트 차트.

**구현 방법**
1. `/stats` 명령어 추가
2. 화자별 발언 세그먼트 수, 주제 분포, 완료율 집계 쿼리
3. 텍스트 막대 그래프 포맷팅 (외부 라이브러리 불필요)

```
/stats
📊 랩미팅 통계 (최근 4회)
발언 비율: 교수님 ████░░ 42%  나 ███░░░ 35%  찬우님 ██░░░░ 23%
주요 주제: 실험결과 > 논문작성 > 향후계획
할일 완료율: ████████░░ 78% (14/18건)
```

---

### 4순위. 할 일 마감 리마인더

**아이디어**: 마감 D-1·D-day에 자동 텔레그램 알림.

**구현 방법**
1. `APScheduler` 추가
2. 매일 09:00 스케줄러 실행
3. 오늘/내일 마감 액션 조회 → chat_id로 알림 전송

```
[자동 알림 - 매일 09:00]
⏰ 오늘 마감 할 일
• 🔴 실험 결과 정리 — 나 (오늘까지)
• 🟡 논문 초안 검토 — 찬우님 (오늘까지)
```

### 5순위. 서브지도교수 페르소나 (장기 과제)

**아이디어**: 누적된 교수님 발언·피드백 데이터로 교수님 말투와 관점을 모사하는 AI 페르소나.
"교수님이라면 내 논문에 뭐라고 하실까?" 질문에 답변.

**평가**
- 구현 용이성 ★★☆☆☆ — 충분한 데이터 누적 필요 (최소 5회 이상)
- 발표 임팩트 ★★★★★
- **2순위(피드백 기록)가 먼저 쌓인 후 진행 권장**

**구현 방법**
1. 교수님 발언 전체를 Few-shot 예시로 구성한 시스템 프롬프트
2. `/ask_prof [질문]` 명령어
3. 실제 피드백 vs AI 피드백 비교 기능

> 발표에서는 **"향후 발전 방향"** 으로 소개하는 전략 권장.
